These docs are designed to separate product requirements from implementation detail:

- The PRD defines the product intent, user experience, scope, and success criteria.
- The technical spec defines the current recommended architecture and data model direction.
- The schema draft turns that data model direction into concrete tables and constraints for discussion.
- The implementation blueprint turns the planning set into an execution-oriented work breakdown before code begins.
- The phases document identifies the planned execution sequence.
- The ideas backlog captures future concepts that should not expand the current scope.

The season-based PRD is the current canonical planning document. The weekly-challenges document set remains available as a future concept that can be mined for follow-on work inside an Explorer season.
